Purchase Approval Example (Tier 2 - Approval)

This example demonstrates a Tier 2 (Approval) process with 2 stages, each containing a role-based obligation. The composition tree has depth 2 (process → stages → obligations), and composition auto-instantiation recursively creates both stage and obligation instances when a process is created.

Overview

Capability Tier: 2 (Approval)
Composition Depth: 2 (process → stages → obligations)
Key Feature: Recursive composition auto-instantiation creates stages AND obligations

Recursive Auto-Instantiation: When a purchase-approval instance is created, the system walks the full composition tree:
  1. Creates Budget Review stage instance + Procurement Review stage instance
  2. For each stage, creates its obligation instance (Budget Approval Obligation, Procurement Approval Obligation)
Total auto-created: 2 stages + 2 obligations = 4 instances, all linked via composition relator instances.

Key Features Demonstrated

  • Recursive Composition: Depth-2 tree (process → stages → obligations)
  • Auto-Instantiation: Both stages AND obligations created automatically
  • FunctionalRoles: Budget Owner and Procurement Manager as obligation assignees
  • Sequential Stages: Budget review must complete before procurement review
  • Tier Detection: Engine finds composition depth 2 (stages + obligations) = Approval tier
  • Obligation Lifecycle: Active → Fulfilled (decision) or Violated (SLO breach)
